38 /** OpenCL kernel to multiply matrices with QASYMM8 data type when only the input matrix RHS (src1) has been reshaped
39  *
40  * @note The input matrix src1 must be reshaped through @ref opencl::kernels::ClGemmReshapeRhsMatrixKernel
41  * @note For fused output stage, only GEMMLowpOutputStageType::QUANTIZE_DOWN_FIXEDPOINT type is supported
42  */

48  /** Initialise the kernel's source and destination.
49  *
50  * @param[in] compile_context The compile context to be used.
51  * @param[in] src0 Input tensor containing the LHS matrix. Data type supported: QASYMM8/QASYMM8_SIGNED
52  * @param[in] src1 Input tensor containing the RHS reshaped matrix. Data type supported: same as @p src0
53  * @param[out] dst Destination tensor. Data type supported: QASYMM8/QASYMM8_SIGNED/S32.
54  * @param[in] gemm_info GEMM information used to retrieve the original dimensions of the input matrices, output stage information and RHS/LHS info.
55  * Only the following values are supported for LHS info:
56  * lhs_info.m0: 2,3,4,5,6,7,8
57  * lhs_info.k0: 2,3,4,8,16
58  * Only the following values are supported for RHS info:
59  * rhs_info.n0: 2,3,4,8,16
60  * rhs_info.k0: same as lhs_info.k0
61  * rhs_info.transpose: true
62  * @param[in] vector_sum_col (Optional) Input row-vector of sums of all the entries in each column of matrix B.
63  * Note: vector_sum_col can be a nullptr in case a_offset = 0. Data type supported: S32
64  * @param[in] vector_sum_row (Optional) Input row-vector of sums of all the entries in each row of matrix A.
65  * Note: vector_sum_row can be a nullptr in case b_offset = 0. Data type supported: S32
66  * @param[in] bias (Optional) Biases tensor. Only shared biases supported and it can be a nullptr if the addition of biases is not required.
67  * Biases are 1D tensor with dimensions [OFM]. Data type supported: S32.
68  * @param[in] output_multipliers (Optional) Output multipliers tensor. In case of per-channel quantization, the number of multipliers must be equal to the number of filters (OFM).
69  * Supported data types: S32.
70  * @param[in] output_shifts (Optional) Output shifts tensor. In case of per-channel quantization, the number of multipliers must be equal to the number of filters (OFM).
71  * Supported data types: S32.
72  */
